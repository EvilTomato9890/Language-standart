# LE

Проверяет одно число меньше либо равно другому или нет

---
## Свойства и функционал 

### Функционал ASM
1. Проверяет 2 числа со стека на NaN, если хотя бы одно из них им является, то бросает на стэк 0. 
2. Потом проверяет эти 2 числа на +- 0, если оба числа это нули, то бросает на стэк 1. 
3. Далее <u>ПОБИТОВО</u> сравнивает эти 2 числа, если они равны, то бросает 1 на стэк, иначе бросает 0.
4. В конце сравнивает эти 2 числа, если 2-ое МЕНЬШЕ 1-ого, то бросает на стэк 1, иначе бросает 0.
>**Вид в ASM**
> ```
> PUSH 1
> PUSH 1
> LE
> ```

### Свойства и особенности
- Если в сравнении участвует NaN, то результат всегда 0

### Примеры

> **Минимальныe примеры:**
> ```
> LE (0 () ()) (1 () ())
> ```
> *Результат: 1*


> **Стандартныe примеры:**
> ```
> LE (PLUS (1 () ())
> 		(1 () ())
> 	 (2 () ())
> ```
> *Результат: 1*
> <br>
> ```
> LE (PLUS (1.05 () ())
> 		(1 () ())
> 	 (2 () ())
> ```
> *Результат: 0*

> **Пограничныe примеры:**
> ```
> LE (NaN () ()) (10 () ())
> ```
> *Результат: 0*
> <br>
> ```
> LE (PLUS (0.1 () ()) 
> 		(0.2 () ()))
> 	 (0.3 () ())
> ```
> *Результат: 0*
> Обычно при округлении 0.1 + 0.2 ≈ 0.30000000000000004, а 0.3 ≈ 0.29999999999999999
